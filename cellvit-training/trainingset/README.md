# cellvit-training/trainingset/

Per-tissue exported training tiles + per-fold training configs.

This folder is **mostly git-ignored** because its bulk content (PNG tiles,
label CSVs, splits, label_map.yaml) is **derived** — every artifact in it
is regenerated by running the pipeline against `data/xenium/`. See
[../README.md](../README.md) for the full pipeline.

## What IS committed

- `trainingset/<tissue>/train_configs/<backbone>/fold_*.yaml` — the
  authoring training configs (with `${CELLVIT_TRAINING_ROOT}/...` tokens).
  These are the source of truth for hyperparameters and class weights; the
  wrapper resolves them via `envsubst` at run time.

## What is NOT committed (regenerated by pipeline)

- `trainingset/<tissue>/train/`        — exported PNG tiles
- `trainingset/<tissue>/train/labels/` — per-tile label CSVs
- `trainingset/<tissue>/test/`         — held-out tiles
- `trainingset/<tissue>/splits/`       — per-fold train/val CSV lists
- `trainingset/<tissue>/cell_labels_*.csv` — per-sample joined labels
  (output of `pipeline/build_cell_labels.py`)
- `trainingset/<tissue>/label_map.yaml` — auto-mirrored from the tissue
  config (output of `pipeline/build_cell_labels.py`)
- `trainingset/<tissue>/train_configs/<backbone>/.fold_*.resolved.yaml` —
  envsubst output materialized at training time
- `trainingset/<tissue>/cache/` — HDF5 dataloader caches (multi-GB per
  tissue; rebuilt automatically on first epoch)

## Regenerating a tissue's training set

```bash
# 1. Per-cell labels + label_map.yaml
python pipeline/build_cell_labels.py --tissue <tissue>

# 2. PNG tiles + per-tile label CSVs (in QuPath, per sample)
#    Open each <sample>_he_image.ome.tif and run qupath/export_tiles.groovy
#    with OUTPUT_ROOT=trainingset/<tissue>/

# 3. Train/val splits
python pipeline/make_splits.py --tissue <tissue>

# 4. Train
bash pipeline/train.sh <tissue>
```

The order is documented in [../README.md](../README.md) §"End-to-end pipeline".
