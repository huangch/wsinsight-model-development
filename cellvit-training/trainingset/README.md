# cellvit-training/trainingset/

Per-tissue exported training tiles + per-fold training configs.

This folder is **mostly git-ignored** because its bulk content (PNG tiles,
label CSVs, splits) is **derived** — every artifact in it is regenerated
by running the pipeline against `data/xenium/`. See
[../README.md](../README.md) for the full pipeline.

## What IS committed

- `trainingset/<tissue>/label_map.yaml` — canonical int ↔ label-name table
  (hand-authored; the QuST/export pipeline and the training config both
  consume this exact file).
- `trainingset/<tissue>/train_configs/<backbone>/fold_*.yaml` — the
  authoring training configs (with `${CELLVIT_TRAINING_ROOT}/...` tokens).
  These are the source of truth for hyperparameters and class weights; the
  wrapper resolves them via `envsubst` at run time.

## What is NOT committed (regenerated by pipeline)

- `trainingset/<tissue>/train/`        — exported PNG tiles
- `trainingset/<tissue>/train/labels/` — per-tile label CSVs
- `trainingset/<tissue>/test/`         — held-out tiles
- `trainingset/<tissue>/splits/`       — per-fold train/val CSV lists
- `trainingset/<tissue>/cache/`        — HDF5 dataloader caches (multi-GB
  per tissue; rebuilt automatically on first epoch)
- `trainingset/<tissue>/train_configs/<backbone>/.fold_*.resolved.yaml` —
  envsubst output materialized at training time

## Regenerating a tissue's training set

```bash
# 1. (one-time) hand-author label_map.yaml for the tissue and add its
#    H&E images to data/qprj/project.qpproj.

# 2. QuST detection + Xenium cluster transfer (headless, batch over project):
QuPath script -s -p data/qprj/project.qpproj \
    cellvit-training/qupath/run_qust_pipeline.groovy

# 3. Remap cluster_id -> label-name on every detection:
QuPath script -s -p data/qprj/project.qpproj \
    -a /abs/path/celltype_assignment_<tissue>_label.csv \
    cellvit-training/qupath/load_mapping.groovy

# 4. Export tiles + per-tile cell CSVs (edit OUTPUT_ROOT at top of the script):
QuPath script -p data/qprj/project.qpproj \
    cellvit-training/qupath/export_tiles.groovy

# 5. Train/val splits
python cellvit-training/pipeline/make_splits.py --tissue <tissue>

# 6. Train + auto-validate
bash cellvit-training/pipeline/train.sh <tissue>
```

The order is documented in [../README.md](../README.md) §"End-to-end
pipeline". To re-apply a fresh `celltype_assignment_*.csv` without
re-running tissue detection + StarDist, use
`cellvit-training/qupath/reset_clusters.groovy` between steps 2 and 3.
